# ページ構成（複数ページ）
共通ヘッダー/フッターあり、モバイルでハンバーガー可。現在ページが分かるアクティブ表示。

- /index.html（トップ）
- /services.html（サービス）
- /works.html（実績）
- /about.html（プロフィール）
- /contact.html（お問い合わせ）
- /privacy.html（プライバシーポリシー）

## 各ページ要件
### トップ（index.html）
- ヒーロー: キャッチコピー、概要、CTA（問い合わせへ）
- サービス概要（2カード）
- 実績ハイライト（ココナラ、ライフコンパスの2カード）
- 依頼の流れ（3〜5ステップ）
- FAQ（よくある質問を5つ程度。例: 料金/進め方/対応範囲/納期/保守）
- CTA（問い合わせ・メール）

### services.html
- サービス詳細（Flutter開発 / 便利ツール）
- 進め方、成果物、よくある依頼例
- よくある依頼例（各サービス3〜4個程度、シンプルで実装が軽いもの中心）:
  - **Flutterアプリ開発**:
    - 業務特化の計算ツール（材料費計算、工数計算、見積もり計算など）
    - データ変換ツール（CSVやJSON形式の変換・整形）
    - QRコード・バーコード生成ツール
    - テキスト処理ツール（文字数カウント、正規表現置換、エンコード変換）
    - シンプルな記録アプリ（日報、作業ログなどローカル保存）
  - **GAS（Google Apps Script）**:
    - スプレッドシートの自動集計・レポート作成
    - フォーム回答の自動整形・通知
    - メール自動送信（定型文、リマインダー）
    - カレンダー連携（予定の自動登録・通知）
    - 複数シート間のデータ連携・同期
  - **iPhoneショートカット**:
    - ワンタップで定型メッセージ送信
    - 位置情報ベースの自動リマインダー
    - 定型フォーマットでのメモ作成
    - 毎日の習慣トラッキング（記録をスプレッドシートへ送信）
    - Wi-Fi接続時の自動アクション設定
- 「要相談」の見せ方を丁寧に（まずヒアリング→提案→見積）

### about.html

- 経歴、資格、技術スタック
- 「寄り添った提案」について具体例を短く（例: 目的整理→要件→最小構成→改善）

### contact.html
- 問い合わせ導線を2つ用意:
  1) フォーム（HTMLで実装）
     - 静的サイトなので送信は以下のどちらかで実装:
       - 第一候補: Formspree等の外部サービス（actionはダミーでOK、差し替え手順をコメントで明記）
       - 第二候補: mailtoリンクで送信（件名/本文テンプレ付き）
  2) メール（mailto）
- フォーム項目:
  - お名前（任意）
  - 連絡先メール（必須）
  - 依頼内容（必須）
  - 予算感（任意）
  - 希望納期（任意）
- 送信後メッセージ（成功/失敗の簡易表示）をJSで実装（外部サービス利用時は最低限）

### privacy.html
- 連絡先情報の取り扱いを中心に、最低限の雛形（日本語）
- 免責の書き方は柔らかく

# 実装指示（重要）
1. 以下のファイル構成で作成してください（必要に応じて追加OK）
   - index.html
   - services.html
   - works.html
   - about.html
   - contact.html
   - privacy.html
   - styles.css
   - script.js（必要なら）
   - assets/favicon.png（簡易でOK）
   - assets/ogp.png（ダミーでOK）
   - sitemap.xml
   - robots.txt
2. 文章は日本語で、過度に盛らず誠実に。読みやすい短文中心。
3. CTAボタンは目立つ位置に複数配置（各ページ上部＋下部）
4. すべてのページで共通のmeta/OGPを適切に設定（ページごとにtitle/descriptionは変える）
5. Lighthouseを意識して軽量に（画像は最小、JSは必要最低限）
6. 外部リンクは target="_blank" + rel="noopener noreferrer"
